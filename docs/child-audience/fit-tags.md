# Fit annotations (menu model)

**Not LEED-style certification.** Parents do not need bronze/silver/gold for a whole festival. They need **per-item labels** — the way food menus mark *gluten-free*, *contains nuts*, *halal*, *dairy-free*: discrete annotations on each activation, zone, and schedule block so families can **filter for intentional design and appropriateness**.

| Food menu | Festival equivalent |
|-----------|---------------------|
| Gluten-free | `quiet` · `sensory-low` |
| Contains nuts | `sensory-loud` · `loud-drums-nearby` (warning) |
| Halal | Cohort fit tags — e.g. `stroller-sightline`, `parallel-play` |
| Vegan | `hands-on` · `passive-viewing-15m` |
| **Combo tags OK** | Lion dance block: `school-age` + `sensory-loud` + `early-exit-ok` |

**Organizer job:** declare tags **on purpose** for each programmed element (map pin, FB Event line, website row) — not one blanket “family-friendly.”

**Parent job:** scan tags against **their** child’s cohort mix (e.g. C0 + SN-A in one stroller).

## C0–C3 + access slugs

Slugs are stable IDs for listings, maps, and Sheets. Display labels can shorten for icons. Two-column summary — see [content patterns](../content-patterns.html#pattern-b--two-column-consolidated-reference-lists).

| Slug | Summary |
|------|---------|

## Usage rules

- Every **activation** gets ≥1 fit tag + any **warning** tags (`sensory-loud`).
- **Absence is not neutral** — untagged block defaults to “unknown fit” in listings, not “all ages.”
- Tags describe **design intent**, not attendance guarantees.
- Same physical zone can carry **multiple tags** (bubble mat: `parallel-play` + `bounded-zone` + `nap-shade`).

**Teen interest/social tags** (`evening-ok`, `peer-social`, `crew-call-HHMM`, etc.) live on [Cohort C4 — teens](cohorts-teens.html).
